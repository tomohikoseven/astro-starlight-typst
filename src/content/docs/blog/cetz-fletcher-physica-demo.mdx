Astro / Starlight と Typst の組み合わせにおける最も強力な利点は、静的なテキストや数式だけでなく、**「プログラマブルなベクターグラフィックス」**と**「高度な科学表記」**をドキュメント内に直接美しく描画できる点にあります。

本記事では、このプロジェクトに組み込まれている3つの強力なライブラリ **CeTZ**、**Fletcher**、**Physica** をすべて使って、Web上に極上の技術文書を作成する方法を解説します。

複雑なダイアグラムコードを個別の `.typ` ファイルに切り出し、Astroコンポーネントとしてインポートする**極めてクリーンで保守性の高い現代的な執筆手法**を紹介します。

## 🛠️ 近代的な執筆アプローチ：コンポーネント化

MDXファイル内に複雑な描画ロジックを直接記述すると、エディタのシンタックスハイライトが効きづらくなったり、Markdownパーサーと `{` や `}` などの記号が干渉してエラーの原因になります。

そのため、本プロジェクトでは**「複雑な図形やグラフは `.typ` ファイルに記述し、MDX内でインポートして呼び出す」**というエレガントな方法を推奨しています。

```mdx
// MDXでのインポート例
import MyDiagram from '../../../components/diagrams/my-diagram.typ';

<MyDiagram />
```

## ⛓️ 3. Fletcher による美しい図式と遷移図

Fletcherは、ノード間のつながりを記述することに特化しています。数式モードと親和性の高いグリッド表記、およびコードによる記述のどちらもサポートしています。

### カテゴリ論の可換図式（同型定理）

数式中に `&`（カラム区切り）と `\`（行区切り）を並べるだけで描画できます（`fletcher-commutative.typ` をインポートしています）。

### リッチなスタイルのフローチャート

## ⚛️ 4. Physica による洗練された物理・数学表記

Physicaは、数学者や物理学者のための記述マクロです。偏微分やブラケット記法など、標準のTypstでは冗長になる記述を極限までシンプルにします。

$$
#import "@preview/physica:0.9.8": *

// 偏微分の等価性
#pdv($f$, $x$, $y$) = #pdv($f$, $y$, $x$) \

// 2次元回転行列とシュレーディンガー方程式で使う表記
#vb($E$) = - nabla V - #pdv(vb($A$), $t$) quad text("and") quad #braket($phi$, $hat(H)$, $psi$) = #expval($hat(H)$, $psi$)
$$
